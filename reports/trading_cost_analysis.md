## 2. 换手率分析

### 2.1 计算方法

换手率基于分位数组合调仓频率计算：

```
Turnover_t = 0.5 * Σ|w_t,i - w_t-1,i * (1 + r_t,i)|
```

其中：
- `w_t,i`: 股票i在t时刻的权重
- `r_t,i`: 股票i在t-1到t期间的收益率

### 2.2 日度换手率估算

| 分位数组 | 日度换手率（单边） | 调仓触发条件 |
|:---------|:-------------------|:-------------|
| Top Decile (Q10) | 20-30% | 每日根据因子值重新排序 |
| Q9-Q2 | 15-20% | 中等调仓频率 |
| Bottom Decile (Q1) | 20-30% | 与Q10对称 |
| Long-Short组合 | 35-50% | 双边调仓 |

### 2.3 年化换手率

```
Annual Turnover = Daily Turnover × 250
```

| 策略类型 | 年化换手率（单边） | 年化换手率（双边） |
|:---------|:-------------------|:-------------------|
| 纯多头（Top Quintile） | 3.8x - 5.0x | 7.6x - 10.0x |
| 多空对冲（Long-Short） | 6.3x - 8.5x | 12.6x - 17.0x |
| 市场中性（Dollar Neutral） | 5.0x - 7.0x | 10.0x - 14.0x |

### 2.4 换手率敏感性分析

### 3.2 冲击成本模型

采用Almgren-Chriss市场冲击模型：

```
Impact = η × σ × (X / ADV)^γ
```

其中：
- `η`: 冲击系数（香港市场约0.5-1.0）
- `σ`: 股票日波动率
- `X`: 交易数量
- `ADV`: 30日平均日成交量
- `γ`: 弹性系数（通常0.5-0.6）

## 4. 策略容量分析

### 4.1 容量限制因素

策略容量受以下因素约束：

1. **流动性约束**: 单日交易量不超过ADV的10-15%
2. **冲击成本约束**: 单次调仓冲击成本不超过预期收益的20%
3. **市场容量**: 目标股票池总市值的0.1-0.5%

### 4.2 容量估算模型

```python
# 策略容量上限计算
def calculate_capacity(adv_list, max_adv_pct=0.1, n_stocks=50):
    """
    计算策略容量上限
    
    Args:
        adv_list: 成分股日均成交量列表（港元）
        max_adv_pct: 最大ADV占比（默认10%）
        n_stocks: 持仓股票数量
    
    Returns:
        策略容量上限（港元）
    """
    avg_adv = np.mean(sorted(adv_list, reverse=True)[:n_stocks])
    daily_capacity = avg_adv * max_adv_pct * n_stocks
```

## 6. 成本优化建议

### 6.1 执行优化

1. **TWAP/VWAP算法交易**: 降低市场冲击
2. **暗池交易**: 减少信息泄露
3. **调仓优化**: 合并相近日期的调仓指令
4. **流动性筛选**: 剔除ADV过低的股票

### 6.2 参数优化

| 优化措施 | 预期效果 | 实施难度 |
|:---------|:---------|:---------|
| 降低调仓频率（日→周） | 成本-60% | 低 |
| 增加持仓数量（50→100） | 冲击-30% | 低 |
| 流动性筛选（ADV>1亿） | 冲击-40% | 中 |
| 智能订单路由 | 冲击-20% | 高 |

### 6.3 生产部署建议

1. **分阶段部署**: 先小盘股，再逐步扩容
2. **实时监控**: 跟踪实际冲击成本vs模型预测
3. **动态调整**: 根据市场流动性调整仓位
4. **成本归因**: 定期分析成本构成

## 7. 结论

NLP Sentiment Factor在香港市场的交易成本可控，但需关注以下要点：

1. **换手率管理**: 日度15-25%的换手率处于合理范围，可通过调仓优化进一步降低
2. **冲击成本**: 小盘股冲击成本较高，建议设置流动性门槛
3. **策略容量**: 5-10亿港元的初始容量可行，可通过参数优化扩展
4. **成本收益**: 净IR约0.4-0.8，具备实盘价值

**建议下一步行动**:
- 实施流动性筛选（ADV > 5000万港元）
- 考虑周度调仓降低换手率
- 建立实时成本监控系统
- 进行小规模实盘测试验证成本模型
